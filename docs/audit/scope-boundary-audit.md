# スコープ境界監査

## 目的

本教材が「理論計算機科学教科書の前提補強」という役割から逸脱しないように、含める内容と含めない内容を明確化する。

## 含める内容

| 領域 | 含める理由 |
|---|---|
| 集合・論理 | 本体教科書の定義・命題・証明を読む基礎 |
| 関数・関係 | オートマトン、遷移、順序、同値性の基礎 |
| 証明技法 | 全章に必要 |
| 漸近記法 | アルゴリズム解析・複雑性理論に必要 |
| 擬似コード・再帰 | アルゴリズムと計算モデルの接続に必要 |
| グラフ・木 | 到達可能性、ネットワーク、状態空間に必要 |
| 組合せ | 数え上げ、下界、確率、情報理論に必要 |
| データ構造 | アルゴリズム章を読む具体例として必要 |
| 形式言語の入口 | オートマトン・計算可能性に進む準備 |
| 確率 | 情報理論、確率的計算、並行性の理解に必要 |
| 数論・代数 | 暗号章に必要 |
| 線形代数の最小限 | 線形符号、状態方程式、行列記法の理解に必要 |
| 並行性の形式モデル | 並行計算章に入る前提 |

## 含めない内容

| 領域 | 理由 |
|---|---|
| 計算可能性理論の本格解説 | 本体教科書の役割 |
| 複雑性理論の本格解説 | 本体教科書の役割 |
| 情報理論の本格解説 | 本体教科書の役割 |
| 暗号プロトコルの詳細 | 本体教科書の役割 |
| 並行計算の不可能性証明 | 本体教科書の役割 |
| 機械学習・深層学習 | 本教材の直接目的外 |
| OS・DB・ネットワーク実務 | ITエンジニア向け実務教材の役割 |
| 競技プログラミングの網羅 | アルゴリズム訓練としては有用だが本教材の主目的外 |

## 判断基準

新規コンテンツを追加する場合、次の質問にすべて答える。

1. 本体教科書のどの章を読む前提になるか。
2. その内容がないと、読者はどの定義・定理・証明で止まるか。
3. 本体教科書の内容を先取りしすぎていないか。
4. 演習または診断で到達確認できるか。
5. 既存章に追記すべきか、新章にすべきか。

上記に答えられない内容は追加しない。
