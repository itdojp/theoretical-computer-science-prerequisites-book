# 表記・用語スタイルガイド

## 目的

教材全体の表記ゆれを減らし、演習・解答・図表・索引の参照性を上げる。

## 基本方針

- 専門用語は初出で日本語と英語を併記してよい。
- 2回目以降は日本語表記を優先する。
- 数式記号は本文・演習・解答で同じ意味に固定する。
- 本体教科書の記法と衝突しない範囲で、標準的なCS・数学記法を使う。

## 用語の標準表記

| 採用表記 | 避ける表記 | 備考 |
|---|---|---|
| 集合 | セット | 数学概念としては「集合」を使う |
| 要素 | エレメント | 初学者向け本文では「要素」 |
| 関数 | 写像 | 第2章では関数を基本表記にする。線形代数では線形写像を許す |
| 終域 | 値域 | `f:A→B` の `B` は終域。像と区別する |
| 像 | 値域 | 実際に到達する出力集合は像 |
| 同値関係 | equivalence relation | 初出後は日本語 |
| 半順序 | partial order | 初出後は日本語 |
| 漸近記法 | asymptotic notation | `O`, `Ω`, `Θ` を含む総称 |
| 擬似コード | 疑似コード | 本教材では「擬似」に統一 |
| 形式言語 | フォーマル言語 | 日本語表記を優先 |
| 空文字列 | 空列 | 形式言語では「空文字列」 |
| 確率変数 | ランダム変数 | 日本語表記を優先 |
| 条件付き確率 | 条件付確率 | 「付き」を送る |
| 合同算術 | 剰余算術 | 第11章では合同算術を採用 |
| 2元体 | GF(2) | 本文では2元体、記号では `F_2` |
| 状態遷移 | state transition | 日本語表記を優先 |
| 非決定性 | nondeterminism | 確率的選択と区別する |

## 記号の標準表記

| 概念 | 採用記号 | 備考 |
|---|---|---|
| 要素関係 | `x∈A` | 前後に空白を入れすぎない |
| 部分集合 | `A⊆B` | 真部分集合は必要時のみ `A⊂B` |
| 含意 | `P→Q` | 証明文脈では `P⇒Q` も可 |
| 全称量化 | `∀x∈S, P(x)` | 否定変形では量化範囲を明示する |
| 存在量化 | `∃x∈S, P(x)` | 同上 |
| 関数型 | `f:A→B` | 本文中では適宜空白を入れてもよい |
| 冪集合 | `P(A)` または `𝒫(A)` | 演習では `P(A)` を優先 |
| Big-O | `O(g(n))` | `O` は大文字 |
| グラフ | `G=(V,E)` | 頂点集合、辺集合の順 |
| アルファベット | `Σ` | 総和記号と文脈で区別する |
| 空文字列 | `ε` | 空集合 `∅` と混同しない |
| 確率 | `Pr[A]` | 条件付き確率は `Pr[A|B]` |
| 期待値 | `E[X]` | 分散は `Var(X)` |
| 合同 | `a≡b (mod n)` | `mod` は丸括弧内 |
| 2元体 | `F_2` | Markdownではコード表記可 |

## 難易度ラベル

| ラベル | 意味 |
|---|---|
| A | 定義・記号・単純計算 |
| B | 複数定義を使う標準問題 |
| C | 証明・設計・概念接続 |
| D | 実装または擬似コード |
| E | 本体教科書への接続 |

問題文では `**[A][定義]**` のように、難易度とカテゴリを並べる。

## 文章スタイル

- 一文を長くしすぎない。
- 「明らか」は避け、使う場合は理由を補う。
- 証明では、仮定、任意に取る対象、示すべき結論を明示する。
- 実装例では、入力前提と境界条件を明示する。
- 解答では、計算結果だけでなく根拠を最低1文添える。
