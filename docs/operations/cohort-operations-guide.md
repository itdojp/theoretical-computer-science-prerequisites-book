# コホート運用ガイド

## 目的

このページは、複数受講者を同時に運用するための手順です。対象は、社内研修、勉強会、ゼミ、オンボーディングです。

コホート運用では、個々の理解度を揃えることではなく、各受講者が本体教科書の該当章へ進める状態を判定することを目的にします。

## 開始前チェック

| 項目 | 確認内容 |
|---|---|
| 対象者 | プログラミング経験、数学経験、本体教科書を読む目的 |
| 期間 | 6週間、10週間、短期集中のどれか |
| 合格基準 | Coreのみか、Standard/Extendedまで含めるか |
| 提出先 | GitHub、社内Git、LMS、共有フォルダのどれか |
| レビュー体制 | 1名あたりのレビュー可能人数 |
| 非同期連絡 | 質問チャネル、期限、再提出ルール |
| 個人情報 | 実名、社員番号、成績の扱い |

## コホート開始手順

```text
1. 対象者を確定する
2. スケジュールを確定する
3. 合格基準を明文化する
4. 提出テンプレートを配布する
5. 前提診断を実施する
6. 診断結果を進捗管理表へ転記する
7. Core / Standard / Extended の受講範囲を個別に決める
8. 初回レビュー会でルールを説明する
```

進捗管理には、[受講者別進捗CSVテンプレート](../forms/learner-progress.csv) と [コホートダッシュボードCSVテンプレート](../forms/cohort-dashboard.csv) を使います。

## 推奨コホートサイズ

| 人数 | 運用方法 |
|---:|---|
| 1〜5名 | 1名の担当者でレビュー可能 |
| 6〜12名 | 週次レビュー会と非同期コメントを併用 |
| 13〜25名 | レビュー担当を2〜3名に分割 |
| 26名以上 | 自動採点できる問題と相互レビューを併用 |

証明問題はレビュー負荷が高いです。人数が多い場合は、全問レビューではなく、代表問題を選んで深く見る方式にします。

## 週次運用サイクル

```text
月曜: 範囲と提出物を告知
火曜〜木曜: 受講者が学習・質問
金曜午前: 提出締切
金曜午後: レビュー一次コメント
翌月曜: 再提出締切、レビュー会
```

各週のレビュー会では、全員の解答を順に読むのではなく、典型的な誤答を2〜3個選び、定義に戻して修正します。

## 進捗ステータス

| ステータス | 意味 | 次のアクション |
|---|---|---|
| Not started | 未着手 | 期限を再確認 |
| Diagnosed | 前提診断済み | 学習ルートを割り当てる |
| In progress | 学習中 | 週次レビュー対象 |
| Needs remediation | 補強が必要 | 弱点別リカバリールートへ戻す |
| Ready for main textbook | 本体教科書へ進行可 | readiness checklist を保存 |
| Deferred | 一時停止 | 再開条件を明文化 |

## レビュー会の進め方

1. 今週の対象定義を1つ選ぶ。
2. 受講者に定義を読み上げさせる。
3. 例を1つ作らせる。
4. 非例を1つ作らせる。
5. 証明または計算を1問だけ板書させる。
6. 誤りを、記号・定義・証明構造・計算のどこにあるか分類する。
7. 次回までの再提出対象を確定する。

レビュー会の記録には、[レビュー会議事録テンプレート](../forms/review-meeting-notes.md) を使います。

## 評価データの扱い

進捗管理表には、必要最小限の情報だけを入れます。

推奨列は次です。

- learner_id
- cohort_id
- diagnostic_score
- core_status
- standard_status
- extended_status
- integrated_test_score
- readiness_status
- reviewer
- next_action

個人評価や人事評価に転用する場合は、事前に利用目的を明示してください。本教材の採点は、理論CS本体に進む準備度を測るものであり、業務能力全体の評価ではありません。

## エスカレーション条件

次の場合は、通常レビューではなく個別対応に切り替えます。

- 2週間連続で提出がない。
- 前提診断の Core 領域が50%未満。
- 証明問題で同じ誤りが3回続く。
- レビューコメントが読まれず、再提出に反映されない。
- 本体教科書へ進ませると明らかに詰まる状態が残っている。

## 終了判定

コホート終了時には、各受講者について以下を保存します。

- 診断結果
- 最終到達ステータス
- 統合テスト結果
- ミニプロジェクト結果
- 本体教科書へ進める章
- 追加補強が必要な章

終了判定は、[本体教科書 readiness checklist](../assessment/main-textbook-readiness.md) を基準にします。
