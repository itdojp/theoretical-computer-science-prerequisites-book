# 教員・研修担当者向け運用ガイド

## 目的

このページは、本教材を個人学習ではなく、社内研修・輪読会・大学ゼミ・オンボーディングで運用する担当者向けのガイドです。

本教材の目的は、理論計算機科学本体に入る前に、読者が次の前提で止まらない状態を作ることです。

- 集合・論理・量化記号
- 関数・関係・同値関係・順序
- 証明技法
- 漸近記法
- 擬似コードと再帰
- グラフ・木・組合せ
- データ構造と基本アルゴリズム
- 文字列・形式言語・オートマトンの入口
- 確率・数論・線形代数・並行性モデルの入口

運用担当者の仕事は、講義で全部を説明することではありません。診断、演習、レビュー、再提出のサイクルを設計し、読者が本体教科書へ進める状態かを判定することです。

## 推奨運用モデル

### 1. 個人非同期型

対象は、自走力の高い受講者です。

```text
前提診断
  ↓
弱点別リカバリールート
  ↓
必要章だけ学習
  ↓
章別レビュー問題
  ↓
統合到達確認テスト
  ↓
本体教科書 readiness checklist
```

担当者は、週1回の提出確認と質問対応だけを行います。

### 2. 週次輪読型

対象は、数学的証明や形式記法に不慣れな受講者です。

```text
事前課題
  ↓
90分セッション
  ↓
小テスト
  ↓
演習提出
  ↓
レビューコメント
  ↓
再提出
```

説明よりも、受講者に定義を読ませ、例・非例・証明を発話させる運用にします。

### 3. 短期集中型

対象は、すでに実務経験があり、CS理論の前提だけを短期補強したい受講者です。

```text
Day 1: 診断、集合・論理・証明
Day 2: O記法、再帰、グラフ
Day 3: 組合せ、データ構造、形式言語
Day 4: 確率、数論、線形代数、並行性
Day 5: 統合テスト、ミニプロジェクト、readiness 判定
```

短期集中型では、全章を深く扱わず、診断で失点した領域に時間を寄せます。

## 役割分担

| 役割 | 責務 |
|---|---|
| 運用責任者 | スケジュール、合格基準、提出期限、評価方針を決める |
| ファシリテータ | 輪読会を進行し、定義・証明・反例の発話を促す |
| レビュー担当 | 演習・ミニプロジェクト・証明答案にコメントする |
| 受講者 | 診断、演習、再提出、振り返りを行う |
| メンテナ | 教材修正、リンク確認、リリース作業を行う |

少人数なら、運用責任者とレビュー担当を兼務できます。10名を超える場合は、レビュー担当を分けるべきです。

## 合格基準

本体教科書への進行判定は、[共通の進行判定ポリシー](../assessment/progression-policy.md)を正本とします。標準合格基準は次です。

| 判定対象 | 合格ライン |
|---|---:|
| 前提診断 | 80問中68問以上 |
| 章別レビュー問題 | 各章70%以上 |
| Core 演習 | 80%以上 |
| Standard 演習 | 70%以上 |
| Extended 演習 | 必要章で70%以上 |
| 統合到達確認テスト | 50問中43問以上、かつ証明問題6問中3問以上 |
| ミニプロジェクト | 20点満点中14点以上 |

ただし、総合得点で証明問題または対象章別 readiness の未達を上書きしてはいけません。本体教科書の第10〜12章へ進む受講者は、Extended の該当章を合格ラインに含めます。

## レビュー方針

レビューでは、正誤だけでなく、次の観点を必ず見ます。

| 観点 | 見る内容 |
|---|---|
| 定義展開 | 用語を感覚で使わず、定義に戻っているか |
| 量化記号 | 全称と存在を取り違えていないか |
| 証明構造 | 仮定、示すべきこと、結論が分離されているか |
| 反例 | 主張が偽のとき、最小の反例を構成できるか |
| 計算量 | O記法を雰囲気ではなく定義または支配項で扱っているか |
| 実装 | 擬似コードとコードの対応が説明されているか |
| 本体接続 | どの本体章の前提になるかを理解しているか |

## 介入基準

次の状態が2回続く受講者には、個別補強を入れます。

- 証明問題で、例を確認しているだけで証明になっていない。
- `∀` と `∃` の否定を毎回誤る。
- 再帰の停止条件を説明できない。
- O記法で定数や低次項の扱いを誤る。
- グラフ探索の「発見済み」と「処理済み」を区別できない。
- DFAの状態遷移を表で追えない。
- 条件付き確率と独立性を混同する。
- 合同算術で逆元の存在条件を無視する。

介入では、受講者に該当章を再読させるだけでは不十分です。必ず、答案を1問選び、定義展開から書き直させます。

## 運用上のアンチパターン

| アンチパターン | 問題 |
|---|---|
| 講義動画のように一方的に説明する | 証明を書く訓練にならない |
| 正解だけを配る | どこで定義を使ったかが残らない |
| Python実装だけで済ませる | 数学的主張の正しさを説明できない |
| 本体教科書へ早く進めすぎる | 第3章以降で記法と証明が負債化する |
| 全員に同じ範囲を強制する | 既習者の時間を浪費し、未習者には速すぎる |
