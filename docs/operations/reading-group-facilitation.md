# レビュー会・輪読会の進行手順

## 基本方針

輪読会では、担当者が説明し続けるのではなく、受講者に次の行動をさせます。

- 定義を読む。
- 記号を日本語に直す。
- 例と非例を作る。
- 小さな証明を書く。
- 擬似コードを1行ずつ追う。
- 誤答を分類する。

理論計算機科学の前提学習では、「分かった気がする」状態が最も危険です。発話、板書、答案レビューで外部化します。

## 90分セッションの標準構成

| 時間 | 内容 |
|---:|---|
| 0〜10分 | 前回の誤答パターン確認 |
| 10〜25分 | 今日の定義を読む |
| 25〜40分 | 例・非例を作る |
| 40〜60分 | 代表問題を板書で解く |
| 60〜75分 | ペアレビューまたは小グループレビュー |
| 75〜85分 | 本体教科書との接続を確認 |
| 85〜90分 | 次回提出物を確定 |

## セッション前の準備

ファシリテータは、事前に次を決めておきます。

- 今日扱う定義を1〜3個に絞る。
- 代表問題を2問選ぶ。
- よくある誤答を想定する。
- 本体教科書のどの章に接続するかを示す。
- 提出物と締切を明示する。

## 発問テンプレート

### 定義確認

```text
この定義で、対象となる集合は何か。
この記号の定義域と値域は何か。
全称量化されている変数はどれか。
存在を示す必要がある対象は何か。
```

### 証明確認

```text
仮定は何か。
示すべき結論は何か。
どの定義を展開すべきか。
この証明は直接証明、対偶証明、背理法、帰納法のどれか。
反例を作るなら、最小の候補は何か。
```

### 擬似コード確認

```text
ループに入る前の状態は何か。
各反復で変わる変数は何か。
不変な性質は何か。
停止条件は何か。
返り値が仕様を満たす理由は何か。
```

## 代表的な進行パターン

### 集合・論理回

1. `A ⊆ B` の定義を読む。
2. `¬∀x P(x)` の変形を全員で書く。
3. 含意の逆・裏・対偶を分類する。
4. 誤答例を示し、どこが誤りかを言語化する。

### 証明回

1. 主張を「仮定」と「結論」に分ける。
2. 直接証明で書く。
3. 対偶証明に書き換える。
4. 帰納法の初期ステップと帰納ステップを分離する。
5. 例示だけの答案を証明に修正する。

### O記法回

1. 定義を読む。
2. `3n^2 + 5n + 1 = O(n^2)` を定義から示す。
3. ループの計算量を数える。
4. 二分探索の `log n` を説明する。
5. 支配項だけの説明で足りる場合と、定義が必要な場合を区別する。

### 形式言語回

1. アルファベット、文字列、言語を区別する。
2. DFAの5つ組を読む。
3. 入力文字列を1文字ずつ処理する。
4. NFAの非決定性をランダムと混同しないように説明する。

## ペアレビュー手順

```text
1. 互いに答案を交換する。
2. 採点せず、まず定義が使われている箇所に印を付ける。
3. 仮定と結論が明示されているかを見る。
4. 量化記号の範囲を確認する。
5. 反例または境界ケースを1つ試す。
6. コメントを「誤り」「不足」「改善提案」に分ける。
```

## 議事録

各回の議事録には、次を残します。

- 扱った章と定義
- 代表問題
- 典型誤答
- 次回までの再提出対象
- 本体教科書への接続
- 未解決質問
